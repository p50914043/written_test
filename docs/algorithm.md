# leetcode算法

## 核心基础算法（必掌握，覆盖 60% 简单 / 中等题）
这类算法是所有题目的基石，新手优先攻克。
### 双指针（Two Pointers）
核心思想：用两个指针（快慢、左右、首尾、前后）遍历数据，将嵌套循环的 O (n²) 复杂度降为 O (n)。
应用场景：
数组 / 字符串：有序数组去重、两数之和（有序版）、三数之和、反转字符串、判断回文、盛最多水的容器。
链表：找中间节点、判断环、删除倒数第 k 个节点、合并两个有序链表。

典型题目：LC 11（盛最多水的容器）、LC 15（三数之和）、LC 19（删除链表倒数第 k 个节点）、LC 26（有序数组去重）。

### 滑动窗口（Sliding Window）
核心思想：用「左 / 右指针」维护一个动态窗口，窗口内是满足条件的子数组 / 子串，通过移动指针动态调整窗口大小，避免重复计算。
应用场景：子数组 / 子串的最值、子串包含所有字符、最长无重复子串、最小覆盖子串。
典型题目：LC 3（最长无重复子串）、LC 76（最小覆盖子串）、LC 209（长度最小的子数组）、LC 438（找到字符串中所有字母异位词）。

### 二分查找（Binary Search）
核心思想：将有序数据集不断二分，每次排除一半元素，将 O (n) 查找降为 O (logn)，核心是「确定边界」和「终止条件」。
应用场景：有序数组查找目标值、找边界（第一个 / 最后一个满足条件的元素）、旋转有序数组、平方根 / 猜数字等数值查找。
典型题目：LC 704（基础二分）、LC 33（搜索旋转排序数组）、LC 34（在排序数组中查找元素的第一个和最后一个位置）、LC 69（x 的平方根）。

### 递归与回溯（Recursion & Backtracking）
核心思想：递归是「自顶向下」分解问题，回溯是「尝试 - 回退」，用于枚举所有可能的解，核心是「剪枝」优化。
应用场景：排列组合、子集、全排列、括号生成、迷宫 / 路径问题、数独 / 八皇后。
典型题目：LC 46（全排列）、LC 78（子集）、LC 22（括号生成）、LC 37（解数独）。

## 数据结构专项算法（高频，覆盖 25% 中等 / 困难题）
算法依赖数据结构，这类题目是 “数据结构 + 基础算法” 的结合。

### 哈希表（Hash Table）
核心思想：用哈希函数映射键值对，实现 O (1) 查找 / 插入，解决「快速查找、计数、去重」问题。
应用场景：两数之和、数组元素计数、判断是否存在重复元素、字母异位词分组。
典型题目：LC 1（两数之和）、LC 49（字母异位词分组）、LC 146（LRU 缓存）、LC 387（字符串中的第一个唯一字符）。

### 栈与队列（Stack & Queue）
核心思想：栈（后进先出）解决「匹配、逆序、临时存储」；队列（先进先出）解决「层级遍历、任务排队」。
应用场景：
栈：括号匹配、有效的括号、每日温度、逆波兰表达式求值。
队列：二叉树层序遍历、滑动窗口最大值（单调队列）、任务调度。
典型题目：LC 20（有效的括号）、LC 739（每日温度）、LC 102（二叉树的层序遍历）、LC 239（滑动窗口最大值）。

### 树与图（Tree & Graph）
树（核心是二叉树）：
核心操作：遍历（前 / 中 / 后序、层序）、递归 / 迭代遍历、构造树、路径求和。
典型题目：LC 94（二叉树中序遍历）、LC 104（二叉树的最大深度）、LC 236（二叉树的最近公共祖先）、LC 105（从前序与中序遍历序列构造二叉树）。
图：
核心操作：遍历（DFS/BFS）、最短路径（Dijkstra/Floyd）、拓扑排序、并查集（解决连通性）。
典型题目：LC 200（岛屿数量）、LC 743（网络延迟时间）、LC 207（课程表）、LC 547（省份数量）。

### 堆（优先队列）
核心思想：基于完全二叉树实现，快速获取最值（O (1) 查最值，O (logn) 插入 / 删除）。
应用场景：topK 问题、数据流的中位数、合并 k 个有序链表、任务调度。
典型题目：LC 215（数组中的第 K 个最大元素）、LC 23（合并 K 个升序链表）、LC 480（滑动窗口中位数）。

## 进阶算法（中等 / 困难题核心，覆盖 10% 高频考点）
### 动态规划（Dynamic Programming, DP）
核心思想：将复杂问题分解为「重叠子问题」，用「状态转移方程」记录子问题解，避免重复计算，核心是「定义状态、找转移方程、确定初始条件」。
应用场景：最值问题（最大子数组和、最长递增子序列）、路径问题、背包问题、打家劫舍、股票买卖。
典型题目：LC 53（最大子数组和）、LC 70（爬楼梯）、LC 1143（最长公共子序列）、LC 198（打家劫舍）、LC 121（买卖股票的最佳时机）。

### 贪心算法（Greedy）
核心思想：每一步选择「局部最优」，最终希望得到「全局最优」，需证明贪心策略的正确性。
应用场景：区间调度、分发饼干、跳跃游戏、加油站、拼接最大数。
典型题目：LC 455（分发饼干）、LC 55（跳跃游戏）、LC 435（无重叠区间）、LC 134（加油站）。

### 并查集（Union-Find）
核心思想：高效解决「动态连通性」问题，支持「合并集合、查找根节点」，带路径压缩和按秩合并优化后接近 O (1)。
应用场景：岛屿数量、省份数量、冗余连接、判断图是否有环。
典型题目：LC 547（省份数量）、LC 684（冗余连接）、LC 990（等式方程的可满足性）。

### 位运算（Bit Manipulation）
核心思想：用二进制位操作（与、或、异或、移位）替代常规运算，高效解决计数、状态压缩问题。
应用场景：只出现一次的数字、位计数、子集（状态压缩）、翻转数位。
典型题目：LC 136（只出现一次的数字）、LC 191（位 1 的个数）、LC 338（比特位计数）。

# 其他常用算法（低频但必要）
分治算法：将问题分解为子问题，解决后合并结果（如归并排序、快速排序、LC 215 分治版）。
模拟算法：按题目规则一步步模拟过程（如 LC 6（Z 字形变换）、LC 415（字符串相加））。
数学算法：质数、公约数、阶乘、排列组合公式（如 LC 204（计数质数）、LC 168（Excel 表列名称））。
总结
优先级排序：双指针 / 滑动窗口 / 二分查找 → 哈希表 / 栈 / 树 → 回溯 / 动态规划 → 贪心 / 并查集 / 位运算，新手先攻克前两类。
核心逻辑：LeetCode 算法题的本质是「数据结构 + 解题模板」，比如二分查找有固定模板，动态规划核心是找状态转移方程。
刷题策略：按算法类型刷（如先刷 10 道双指针题），而非按题号刷，更容易形成解题思维。
