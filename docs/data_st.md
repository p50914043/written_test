# 数组
数组：一种固定大小的线性表，数组的元素按索引顺序存储，索引从 0 开始。
# 链表
链表：一种不固定大小的线性表，链表元素不按索引顺序存储，而是通过指针连接。
# 栈
栈是一种 Last In First Out（LIFO）的数据结构，即后进先出。
# 队列
队列是一种 First In First Out（FIFO）的数据结构，即先进先出。
# 堆
堆：一种特殊类型的树结构，满足以下条件：
1. 每个节点最多有 2 个子节点；
2. 堆是完全二叉树（即除了最后一层，其他层的节点都满，且最后一层的节点都靠左对齐）；
3. 堆中每个节点的值都大于等于（或小于等于）其子节点的值（最大堆或最小堆）。

二叉搜索树、平衡树、红黑树

# B树、B+树
B树（B-Tree）：一种**平衡树**，核心思想是**将数据存储在磁盘文件中，并利用索引来快速查找数据**。B树是**平衡二叉树**的扩展，它将索引节点和数据节点存储在磁盘文件中，并利用索引来快速查找数据。B树通常用于存储大量数据，并支持快速查找和插入操作。

# 跳表
跳表（Skip List）是一种基于**有序链表**扩展的高效数据结构，核心思想是通过 “分层索引” 为有序链表建立多级 “快捷通道”，从而将普通有序链表 O (n) 的查询效率提升到平均 O (log n)（与红黑树 / AVL 树同级），且实现复杂度远低于平衡树。
注意：是有序链表
通俗解释：普通有序链表像一条只有地面道路的街道，只能逐个走；跳表则给街道加了多层天桥（索引层），可以通过天桥跳过大部分节点，快速到达目标位置。

以有序链表 1 → 3 → 5 → 7 → 9 → 11 为例，跳表的分层结构如下：
底层（Level 0）：完整的有序链表（所有节点都在这一层）；
Level 1：每隔 1 个节点建立索引（如 1 → 5 → 9）；
Level 2：每隔 2 个节点建立索引（如 1 → 9）；
顶层：索引节点最少（通常只有 1 个起始节点）。
